title: Reference images
description: Optional reference images for palette and composition hints

---

You can attach an optional reference image to `POST /api/v1/logos` to steer palette and composition. Reference data is used **only for that request** and is **not saved** to your Logo Lattice account.

## Request fields

| Field | Required | Description |
| --- | --- | --- |
| `referenceImageBase64` | No | Base64-encoded image bytes |
| `referenceMimeType` | Yes, when base64 is set | `image/png`, `image/jpeg`, or `image/webp` |

## Limits

| Limit | Value |
| --- | --- |
| Max decoded size | **4 MB** |
| Supported types | PNG, JPEG, WebP |

## Privacy

Reference images are sent to the model for the single generation request. They are not stored as assets on your account and do not appear in the saved cloud project unless the generated logo incorporates similar styling.

## Prompt guidance

When using a reference image, say in `prompt` what to borrow and what to simplify—for example palette and silhouette, not every detail of the source image.

```bash
curl -X POST "https://logolattice.com/api/v1/logos" \
  -H "Authorization: Bearer $LOGO_LATTICE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "Match the teal palette and triangular silhouette, simplify to an isometric lattice mark",
    "referenceImageBase64": "'$(base64 -i reference.png | tr -d '\n')'",
    "referenceMimeType": "image/png"
  }'
```

On macOS, `base64 -i` reads a file; on Linux you may use `base64 -w 0 reference.png` instead of `tr -d '\n'`.

See [Best practices](/api-reference/studio-logo-api/best-practices) for prompt patterns that work well with references.
