## Shared credit pool

Studio AI credits are **shared** across:

- `POST /api/v1/logos` (Describe / `openrouter` and SVG paths via `provider`)
- `POST /api/v1/logos/svg-paths`
- In-app **Describe**
- In-app **Export → AI stylise**

## Monthly allowance

| Concept | Detail |
| ------- | ------ |
| Free monthly cap | **10** generations per **UTC calendar month** |
| Top-ups | Bonus credits stack and **do not expire** |
| Check balance | `GET /api/v1/logos/usage` (no credit cost) |

Check your balance anytime with [`GET /api/v1/logos/usage`](/api-reference/describe-api/get-usage).

## Credit cost per operation

| Operation | Credits |
| --------- | ------- |
| Describe / default `POST /api/v1/logos` (`openrouter`) | **1** per success |
| SVG paths `quality: "standard"` | **1** |
| SVG paths `quality: "enhanced"` | **2** |
| `GET /api/v1/logos/usage` | **0** |

Failed generations (upstream or API error after a failed attempt) generally **do not** consume a credit, same as in-editor Describe.

## When you are out of credits

When `remaining` is `0`, `POST` returns **429** with `code: "limit_exceeded"` and a `usage` object until either:

- The monthly pool resets at `resetsAt` (UTC month boundary), or
- You purchase a credit top-up in Studio.

Example message:

```json
{
  "error": "You have no AI generations left. Top up for 30 more, or wait until your free quota resets.",
  "code": "limit_exceeded",
  "usage": { "monthlyUsed": 10, "monthlyRemaining": 0, "remaining": 0 }
}
```

## Usage fields

Successful generations and `GET /usage` return the same usage shape. Prefer `monthlyLimit` and `monthlyRemaining` for monthly allowance UI; use `remaining` for “can I generate now?” checks.

<Info>
  The `limit` field is **deprecated** - it is an alias of `monthlyLimit`.
</Info>
