Generate a logo from a text brief. Returns SVG, a cloud `projectId`, and an `editUrl` to continue in the editor.

## Request

<ParamField header="Authorization" type="string" required>
  `Bearer <api_key>` - Studio API key (`ll_live_…`).
</ParamField>

<ParamField header="Content-Type" type="string" required>
  `application/json`
</ParamField>

### Body

<ParamField body="prompt" type="string" required>
  Logo brief after trimming. **10–800 characters.** Describe style, subject, colours, and grid feel (for example isometric, minimal facets).
</ParamField>

<ParamField body="referenceImageBase64" type="string">
  Base64-encoded reference image for palette or composition inspiration. Max **4 MB** decoded. Not saved to your account. See [Reference images](/api-reference/describe-api/reference-images).
</ParamField>

<ParamField body="referenceMimeType" type="string">
  Required when `referenceImageBase64` is set. One of: `image/png`, `image/jpeg`, `image/webp`.
</ParamField>

<ParamField body="brandName" type="string">
  Optional label for the palette in the model prompt (for example your Studio brand name). **1–64 characters** after trim. Ignored if empty or over 64 characters. Has no effect unless `brandColors` and/or `referenceColors` are provided.
</ParamField>

<ParamField body="brandColors" type="string[]">
  Optional full brand palette. Up to **5** unique valid hex colours after normalization. See [Colour format](#colour-format) below.
</ParamField>

<ParamField body="referenceColors" type="string[]">
  Optional colours the model should emphasize in fills and accents. Up to **5** unique valid hex colours; same format rules as `brandColors`.
</ParamField>

### Colour format

| Rule | Behaviour |
| --- | --- |
| Valid formats | `#RGB`, `#RRGGBB`, or without `#` (for example `0d9488`) |
| Normalization | Server expands 3-digit hex and stores lowercase `#rrggbb` |
| Duplicates | Removed; order preserved for the first occurrence |
| Array length | More than 5 entries → only the first 5 unique valid colours are kept |
| Invalid entry | **400** with Zod `fieldErrors` on `brandColors` or `referenceColors`: `"Invalid hex colour."` |

### How brand fields affect generation

Optional colour fields are appended to the user message as a **Colour direction** block, the same logic as **Describe** in the editor:

| Request | Prompt block |
| --- | --- |
| `brandName` + `brandColors` | `Brand palette ({brandName}): #..., #...` |
| `brandColors` only | `Brand palette: #..., #...` |
| `referenceColors` | `Use these hex colours prominently in fills and accents unless the brief clearly conflicts: ...` |
| `brandColors` without `referenceColors` | Also adds: prefer the brand palette unless the brief clearly conflicts |
| `brandName` alone (no colour arrays) | No colour block is added |

Combine `referenceImageBase64` with brand fields when you want both visual reference and explicit palette control.

### Brand colours in the API

The Logo API uses the same colour-direction logic as **Describe** in the editor. The API does **not** load brands from your Studio account. Pass `brandName`, `brandColors`, and optionally `referenceColors` explicitly. Integrations should read colours from their own config or from Studio brand data they already store. There is no `GET /studio/brands` on the Logo API.

| Describe UI | API body |
| --- | --- |
| No brand | Omit `brandName`, `brandColors`, `referenceColors` |
| Brand selected, colour reference **None** | Omit all three (or omit `referenceColors` and do not send `brandColors` if you want no palette) |
| **All brand colours** | `brandName`, `brandColors` = full palette, `referenceColors` = same as `brandColors` |
| Single swatch | `brandName`, `brandColors` = full palette, `referenceColors` = `[that hex]` |

Palette context only (softer colour guidance, no `referenceColors`):

```json
{
  "prompt": "Geometric fox head, warm palette",
  "brandName": "Acme",
  "brandColors": ["#f59e0b", "#1c1917", "#ffffff"]
}
```

<Note>The response body (`svg`, `editUrl`, `projectId`, `usage`) is unchanged when you add brand colour fields.</Note>

<?php

$apiKey = getenv('LOGO_LATTICE_API_KEY');

    CURLOPT_POST => true,
    CURLOPT_RETURNTRANSFER => true,
    CURLOPT_HTTPHEADER => [
        'Authorization: Bearer ' . $apiKey,
        'Content-Type: application/json',
    ],

$raw = curl_exec($ch);

## Response

<ResponseField name="svg" type="string" required>
  Full SVG document with clean paths. Transparent artboard; suitable for web, print, and design tools. Server-side export. Web fonts in text logos may differ slightly from the browser editor when
  custom fonts were used.
</ResponseField>

<ResponseField name="editUrl" type="string" required>
  URL to open the project in Logo Lattice. Requires the API key owner to be signed in. Not a public anonymous editor link.
</ResponseField>

<ResponseField name="projectId" type="string" required>
  Cloud project ID (also appears as the `project` query parameter in `editUrl`).
</ResponseField>

<ResponseField name="usage" type="object" required>
  Remaining AI credits after this generation. Monthly limit resets at the UTC month boundary (`resetsAt`).
</ResponseField>

<ResponseField name="usage.monthlyUsed" type="integer">
  Credits used this calendar month (UTC).
</ResponseField>

<ResponseField name="usage.monthlyLimit" type="integer">
  Monthly allowance (Studio includes **10** per month, shared with Describe and AI stylise).
</ResponseField>

<ResponseField name="usage.monthlyRemaining" type="integer">
  Credits left in the monthly pool before bonus credits.
</ResponseField>

<ResponseField name="usage.bonusCredits" type="integer">
  Extra credits from top-ups.
</ResponseField>

<ResponseField name="usage.remaining" type="integer">
  Total credits available (`monthlyRemaining` + bonus).
</ResponseField>

<ResponseField name="usage.limit" type="integer">
  Deprecated - prefer `monthlyLimit` for display.
</ResponseField>

<ResponseField name="usage.resetsAt" type="string">
  ISO 8601 timestamp when the monthly pool resets (UTC month boundary).
</ResponseField>

## HTTP status codes

| Status | When                                                                                     |
| ------ | ---------------------------------------------------------------------------------------- |
| `200`  | Success                                                                                  |
| `400`  | Invalid JSON, validation failed (prompt, reference image, invalid hex in `brandColors`/`referenceColors`, `brandName` over 64 chars), parse error, empty model result, or cloud project limit |
| `401`  | Missing, invalid, or revoked Bearer token                                                |
| `403`  | Account is not Studio                                                                    |
| `429`  | No AI credits remaining                                                                  |
| `502`  | Upstream model failure                                                                   |
| `503`  | AI not configured on server                                                              |

See [Errors](/api-reference/describe-api/errors) for `code` values and examples.
