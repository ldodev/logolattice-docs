Every `/api/v1/logos` endpoint requires a Studio API key in the `Authorization` header:

```http
Authorization: Bearer ll_live_…
Content-Type: application/json
```

(Post routes only need `Content-Type`; `GET /usage` needs `Authorization` only.)

## Key format

| Item | Value |
| ---- | ----- |
| Header | `Authorization: Bearer ll_live_…` |
| Key prefix | `ll_live_` (full secret shown **once** when created) |
| Max active keys | **5** per account |
| Subscription | **Studio** required |

<Warning>
  Never commit API keys to source control. Store keys in environment variables or a secrets manager (for example `LOGO_LATTICE_API_KEY`).
</Warning>

## Create an API key

<Steps>
  <Step title="Sign in to Studio">
    Open [Logo Lattice](https://logolattice.com) and sign in with a Studio account.
  </Step>
  <Step title="Open API keys">
    Go to **Studio → API keys** at [`/studio/api-keys`](https://logolattice.com/studio/api-keys).
  </Step>
  <Step title="Create and copy">
    Create a named key and copy the full secret immediately. Logo Lattice stores only a SHA-256 hash; the secret cannot be retrieved later.
  </Step>
  <Step title="Store securely">
    Save the key in your deployment secrets or `.env` file (never in git).
  </Step>
</Steps>

## Auth failures

| HTTP | `code` | When |
| ---- | ------ | ---- |
| `401` | `invalid_key` | Missing or malformed Bearer header, wrong prefix, or unknown key |
| `401` | `revoked` | Key was revoked |
| `403` | `not_premium` | Account no longer has Studio |

## Example

```bash
export LOGO_LATTICE_API_KEY="ll_live_YOUR_SECRET_HERE"

curl "https://logolattice.com/api/v1/logos/usage" \
  -H "Authorization: Bearer $LOGO_LATTICE_API_KEY"
```

Revoke keys anytime from **Studio → API keys**. Update integrations before revoking keys that are still in use.
