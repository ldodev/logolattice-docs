Strong prompts produce marks that fit Logo Lattice’s grid-based vector style. Treat the `prompt` field like a creative brief, not marketing copy.

## Focus on one mark

Describe **one symbol or mark**, not a full scene, UI mockup, or long brand paragraph.

<Tip>**Good:** Isometric teal mountain, minimal facets, square grid, single icon</Tip>

<Warning>**Avoid:** A complete landing page hero with text, buttons, and background scenery</Warning>

## Specify grid and detail

Mention:

- **Grid type** - square vs isometric
- **Palette** - named colours or mood (teal, monochrome, warm earth tones)
- **Detail level** - minimal facets, bold blocks, fine lattice lines

Example:

```
Isometric amber fox head, 6–8 facets, warm palette, clean silhouette for app icon
```

## Pass brand colours explicitly

The API does not load Studio brands by ID. When you have a customer or product palette, send `brandName`, `brandColors`, and optionally `referenceColors` as hex values, the same fields the editor sends after a user picks a brand and colour reference.

- **All brand colours** (editor): put the full palette in both `brandColors` and `referenceColors`.
- **Single swatch** (editor): full palette in `brandColors`, one hex in `referenceColors`.
- **Softer guidance**: send only `brandColors` (and optional `brandName`) without `referenceColors` so the model prefers the palette unless the brief conflicts.

Agents and MCP integrations can map palette hex from customer brand guidelines into these fields.

## Use reference images deliberately

When you send `referenceImageBase64`, state in the prompt:

- What to **borrow** (palette, silhouette, proportions)
- What to **simplify** (drop texture, reduce to lattice geometry)

Example:

```
Match the navy and gold palette from the reference; simplify to a square-grid lettermark, no gradients
```

## Length

After trimming, keep prompts between **10 and 800 characters**. Shorter, specific briefs usually outperform long unstructured text.

## Choose Describe vs SVG paths

| Goal | Approach |
| ---- | -------- |
| Continue editing in Logo Lattice with a cloud project | Default `POST /api/v1/logos` (Describe) |
| Path-only SVG for your own pipeline | `POST /api/v1/logos/svg-paths` or `"provider": "svg-paths"` |
| Higher-fidelity paths (2 credits) | `"quality": "enhanced"` on SVG paths |

## Test in the editor first

Default Describe uses the same model as **Describe** in the editor. If a brief works well there, it will likely work via the API.
